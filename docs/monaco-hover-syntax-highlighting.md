# Syntax-Highlighted Hover Tooltips in Monaco Editor

Monaco's hover widget renders markdown from `registerHoverProvider`, but **does not syntax-highlight fenced code blocks** in standalone mode. This doc explains how LearnTensors adds full syntax highlighting (including semantic tokens) to hover tooltips.

## The Problem

Monaco's hover widget renders `IMarkdownString` content. When a hover provider returns markdown with fenced code blocks:

```typescript
return {
  contents: [{ value: "```python\ndef foo(x: int) -> int\n```" }],
};
```

The code block renders as plain monospace text with no colors.

## Why It's Hard

Three obstacles prevent straightforward approaches:

### 1. `monaco.editor.colorize()` uses CSS classes, not inline styles

`colorize()` returns HTML like `<span class="mtk3">def</span>`. These `mtk*` classes are generated by Monaco's theme engine and work in the editor, but...

### 2. Monaco's DOMPurify sanitizer strips most class attributes

The hover widget runs all HTML through DOMPurify. The sanitizer only preserves `class` on `<span>` if it matches `codicon codicon-*`. All other classes are removed, so `mtk3` gets stripped.

### 3. The sanitizer only allows hex colors in `style` attributes

Even with `supportHtml: true`, the `style` attribute on `<span>` elements is validated against:

```
/^(color:(#[0-9a-fA-F]+|var\(--vscode-...\));)?...$/
```

So `style="color:rgb(86, 156, 214)"` is stripped, but `style="color:#569cd6;"` is kept. The trailing semicolon is also required.

## The Solution

The approach has two layers: Monarch tokenizer for base syntax colors, and semantic tokens for rich type-aware colors.

### Step 1: Parse Fenced Code Blocks from Hover Markdown

The hover provider returns markdown from the language server. Extract fenced code blocks and process them separately:

```typescript
async function colorizeHoverMarkdown(md: string): Promise<string> {
  const parts: string[] = [];
  const codeBlockRe = /```(\w*)\n([\s\S]*?)```/g;
  let last = 0;
  for (const m of md.matchAll(codeBlockRe)) {
    parts.push(md.slice(last, m.index!));
    const code = m[2].trimEnd();
    parts.push(`<pre>${colorizeCode(code)}</pre>`);
    last = m.index! + m[0].length;
  }
  if (parts.length === 0) return md;
  parts.push(md.slice(last));
  return parts.join("");
}
```

Wrapping in `<pre>` preserves whitespace (indentation). Both `<pre>` and `<span>` are in Monaco's allowed HTML tags list.

### Step 2: Build a Character-Level Color Array

For each line, assign a color to every character position. Start with Monarch tokens as the base layer:

```typescript
const monarchLines = monaco.editor.tokenize(code, "python");

for (let i = 0; i < codeLines.length; i++) {
  const line = codeLines[i];
  const charColors: string[] = new Array(line.length).fill(defaultFg);
  const mTokens = monarchLines[i] || [];

  for (let t = 0; t < mTokens.length; t++) {
    const token = mTokens[t];
    const nextOffset = t + 1 < mTokens.length
      ? mTokens[t + 1].offset : line.length;
    const color = monarchColorMap[token.type]; // e.g. "keyword.python" -> "#569cd6"
    if (color) {
      for (let c = token.offset; c < nextOffset; c++) {
        charColors[c] = color;
      }
    }
  }
}
```

`monaco.editor.tokenize()` returns token type strings like `"keyword.python"`, `"identifier.python"`, etc. You need to maintain a manual mapping from these type strings to hex colors matching your theme.

### Step 3: Overlay Semantic Tokens (Optional but Recommended)

Monarch tokenization only gives basic syntax colors. Semantic tokens from a language server distinguish functions, parameters, classes, variables, etc. If you have a language server or type checker (like ty for Python), get semantic tokens for the hover code:

```typescript
// Open a temp file with the hover code in your language server workspace
const tempCode = importPrefix + hoverCode;
workspace.updateFile(tempHandle, tempCode);
const semTokens = workspace.semanticTokens(tempHandle);

// Overlay semantic colors (higher priority than Monarch)
for (const token of semTokens) {
  const color = semanticColorMap.get(tokenKindName);
  if (color) {
    for (let c = token.start; c < token.end; c++) {
      charColors[c] = color;
    }
  }
}
```

**Key detail for type resolution:** Hover markdown often uses abbreviated type names (e.g. `Tensor` instead of `torch.Tensor`). For the language server to resolve these in the temp file, prepend the source file's imports _plus_ wildcard imports for bare `import X` statements:

```typescript
// Source has: import torch
// Hover shows: Tensor (not torch.Tensor)
// Temp file needs: import torch + from torch import *
for (const line of sourceLines) {
  const bare = line.match(/^\s*import (\S+)/);
  if (bare) {
    importLines.push(line);
    importLines.push(`from ${bare[1]} import *`);
  }
}
```

### Step 4: Build HTML with Inline Hex Colors

Group consecutive same-color characters into spans with inline styles:

```typescript
let html = "";
let spanStart = 0;
for (let c = 1; c <= line.length; c++) {
  if (c === line.length || charColors[c] !== charColors[spanStart]) {
    html += `<span style="color:${charColors[spanStart]};">`
      + escapeHtml(line.slice(spanStart, c))
      + "</span>";
    spanStart = c;
  }
}
```

Colors **must** be hex format with a trailing semicolon (`color:#569cd6;`). RGB format is rejected by the sanitizer.

### Step 5: Return with `supportHtml: true`

```typescript
monaco.languages.registerHoverProvider("python", {
  async provideHover(_model, position) {
    const result = languageServer.hover(position);
    if (!result) return null;
    const colorized = await colorizeHoverMarkdown(result.markdown);
    return {
      contents: [{ value: colorized, supportHtml: true, isTrusted: true }],
      range: { ... },
    };
  },
});
```

Both `supportHtml` and `isTrusted` must be `true`. The provider can be `async` (Monaco accepts `Thenable` returns).

## Reading Monarch Token Colors at Runtime

If you want to read the actual `mtk*` class colors from the active theme instead of hardcoding them:

```typescript
function resolveMonarchColors(): Map<string, string> {
  const map = new Map<string, string>();
  for (const sheet of document.styleSheets) {
    try {
      for (const rule of sheet.cssRules) {
        if (rule instanceof CSSStyleRule) {
          const m = rule.selectorText.match(/^\.(mtk\d+)$/);
          if (m && rule.style.color) {
            map.set(m[1], rgbToHex(rule.style.color));
          }
        }
      }
    } catch {} // skip cross-origin sheets
  }
  return map;
}
```

Note that `rule.style.color` returns `rgb()` format even if the original CSS uses hex, so you need to convert:

```typescript
function rgbToHex(rgb: string): string {
  const m = rgb.match(/rgb\(\s*(\d+),\s*(\d+),\s*(\d+)\s*\)/);
  if (!m) return rgb;
  const hex = (n: string) => parseInt(n).toString(16).padStart(2, "0");
  return `#${hex(m[1])}${hex(m[2])}${hex(m[3])}`;
}
```

## Summary

| Layer | Source | What it colors | Priority |
|-------|--------|----------------|----------|
| Monarch tokenizer | `monaco.editor.tokenize()` | Keywords, operators, strings, numbers, delimiters | Base |
| Semantic tokens | Language server / type checker | Functions, parameters, classes, variables, decorators | Override |

The key constraints to remember:
- Use **hex colors** with **trailing semicolons** in `style` attributes
- Wrap code in `<pre>` to preserve whitespace
- Set both `supportHtml: true` and `isTrusted: true`
- Don't use CSS classes — they get stripped by DOMPurify
- Don't use `rgb()` colors — they get stripped by the style validator
